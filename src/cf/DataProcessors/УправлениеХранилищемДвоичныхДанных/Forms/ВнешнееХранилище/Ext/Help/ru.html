Управление внешним хранилищем двоичных данных

Диалог предназначен для настройки выбранного внешнего хранилища двоичных данных.

Общие настройки

  • Имя - Уникальное название внешнего хранилища двоичных данных, не более 32 символов.
  • Идентификатор доступа - Идентификатор доступа (AccessId) к внешнему хранилищу двоичных данных.
  • Ключ доступа - Секретный ключ доступа (SecretKey) к внешнему хранилищу двоичных данных.
  • Разрешить запись - Если установлен, все новые данные, для которых выполнена настройка, будут записываться в хранилище. Данные, которые уже находятся в хранилище, будут получаться из него и будут доступны для пользователя.
  • Хранилище по умолчанию  - Если установлен, то все новые данные, для которых не была выполнена настройка, будут записываться в хранилище. Данные, которые уже находятся в хранилище, будут получаться из него и будут доступны для пользователя.
  • Минимальный размер данных (байт) - отображает текущее значение размера данных, при превышении которого данные будут записаны в хранилище.

После изменения необходимых полей необходимо нажать кнопку "ОК" для обновления настроек внешнего хранилища, либо "Отмена" для закрытия формы и возврата к форме выбора хранилища.

Внимание! Для настройки хранилища двоичных данных требуется монопольный режим работы с информационной базой.

Очистка хранилища

Данная закладка предназначена для запуска очистки неиспользуемого места внешнего хранилища двоичных данных. Для этого необходимо выбрать дату и время в поле "Очистить данные до даты", после чего нажать кнопку "Очистить".
