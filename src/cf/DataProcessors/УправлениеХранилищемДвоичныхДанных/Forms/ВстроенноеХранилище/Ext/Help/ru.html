Управление хранилищем двоичных данных

Диалог предназначен для настройки встроенного хранилища двоичных данных.

Общие настройки

  • Хранилище двоичных данных - управляет режимом использования хранилища двоичных данных. После отключения хранилища все ранее записанные в него данные становятся недоступными для чтения.
  • Разрешить запись - Если установлен, все новые данные, для которых выполнена настройка, будут записываться в хранилище. Данные, которые уже находятся в хранилище, будут получаться из него и будут доступны для пользователя.
  • Хранилище по умолчанию  - Если установлен, то все новые данные, для которых не была выполнена настройка, будут записываться в хранилище. Данные, которые уже находятся в хранилище, будут получаться из него и будут доступны для пользователя.
  • Минимальный размер данных (байт) - отображает текущее значение размера данных, при превышении которого данные будут записаны в хранилище. Чтобы установить другой размер, нажмите кнопку Изменить.

После изменения необходимых полей необходимо нажать кнопку "ОК" для обновления настроек внешнего хранилища, либо "Отмена" для закрытия формы и возврата к форме выбора хранилища.

Внимание! Для настройки хранилища двоичных данных требуется монопольный режим работы с информационной базой.

Создание резервной копии

Данная закладка предназначена для настройки и создания резервной копии хранилища двоичных данных.

Выберите тип создаваемой копии:

  • Полная - сохраняет все содержимое хранилища двоичных данных.
  • Дифференциальная - сохраняет изменения, которые произошли с данными с момента начала предыдущего полного резервного копирования. Для выполнения дифференциального резервного копирования требуется наличие одной полной резервной копии.

Создание резервной копии зависит от выбранного типа:
  • Для создания полной копии требуется указать Имя файла полной резервной копии.
  • Для создания дифференциальной копии требуется указать Имя файла полной резервной копии (от которой будет создаваться дифференциальная резервная копия) и Имя файла дифференциальной копии. 

При нажатии кнопки Создать будет сформирован файл c расширением:

  • .sbf - для полной резервной копии,
  • .sbd - для дифференциальной резервной копии.

Учетная запись операционной системы от имени которой запущен сервер 1С, обслуживающий сервис хранилища двоичных данных, должна иметь доступ к файлам создаваемых или восстанавливаемых резервных копий.

Восстановление из резервной копии

Данная закладка предназначена для восстановления хранилища двоичных данных из резервной копии.

Выберите тип разервной копии для восстановления:

  • Полная - сохраняет все содержимое хранилища двоичных данных.
  • Дифференциальная - сохраняет изменения, которые произошли с данными с момента начала предыдущего полного резервного копирования. Для выполнения дифференциального резервного копирования требуется наличие одной полной резервной копии.

Восстановление данных из резервной копии зависит от выбранного типа:

  • При выборе полной резервной копии достаточно указать Имя файла полной резервной копии.
  • При выборе дифференциальной копии требуется указать Имя файла полной резервной копии и Имя файла дифференциальной копии (должен быть создан на основании указанного файла полной копии). 

Для восстановления данных из указанной копии нажмите кнопку Восстановить.
Учетная запись операционной системы от имени которой запущен сервер 1С, обслуживающий сервис хранилища двоичных данных, должна иметь доступ к файлам создаваемых или восстанавливаемых резервных копий.

Внимание! Для восстановления хранилища двоичных данных из резервной копии требуется монопольный режим работы с информационной базой.

Информация о хранилище

Различная статистическая информация о хранилище двоичных данных:

  • Объем хранимых данных

    Объем хранилища двоичных данных в байтах, без учета сжатия. 
    Включает в себя длину актуальных, а также помеченных для удаления данных. Файлы метаданных хранилища не учитываются.

  • Объем данных для отложенного удаления

    Объем данных, помеченных для удаления, но еще физически не удаленных, в байтах.

  • Количество элементов двоичных данных в хранилище

    Включает в себя объекты актуальных, а также помеченных для удаления данных.

  • Количество элементов двоичных данных для отложенного удаления

    Количество объектов, помеченных для удаления, но еще физически не удаленных.

  • Объем хранимых данных с учетом дедупликации

    Размер данных в хранилища в данных на текущий момент с учетом дедупликации.

  • Количество дедуплицированных объектов двоичных данных

    Количество дедуплицированных элементов в хранилище двоичных данных.

  • Дата последней очистки неиспользуемых данных

    Наибольшее дата/время, на которое производилась очистка хранилища двоичных данных.

  • Обновить

    По нажатию этой кнопки обновляются данные информации хранилища двоичных данных.

Очистка хранилища

Физически очищает удаленные данные из хранилища двоичных данных до указанной даты.

  • Выполнять очистку автоматически

    Указывает, будет ли очистка хранилища двоичных данных вызываться автоматически после окончания создания резервной копии. 
    Если автоматическая очистка выключена, то будет возможность выполнить очистку вручную путем нажатия кнопки "Очистить".

  • Очистить данные до указанной даты

    Указывает дату, до которой будут физически очищены из хранилища удаленные данные.

  • Очистить

    Запускает очистку хранилища (физическое удаление помеченных на удаление данных хранилища). 
    Если выбранная дата очистки будет раньше, чем дата с последней очистки, то очистка будет сразу успешно завершена. Время, на которое была сделана последняя очистка, не изменится.

  • Автоматически очищать хранилище двоичных данных, сохраняя удаленные данные за последние ХХ дней

    Данные, удаленные в течение этого периода, не будут физически очищены из хранилища при очередной чистке.

  • Применить

    Применить настройку периода автоматического очищения удаленных данных из хранилища двоичных данных.
